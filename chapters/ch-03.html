What a Server Actually Does

The listen-receive-route-process-respond loop and the middleware pipeline that handles every request

~10 min read

The Server Loop

A server follows a simple loop: listen, accept, parse, route, process, respond, then go back to listen. This cycle runs on every server, from a Raspberry Pi to a Google datacenter.

  • Listen — The OS reserves a port for the process
  • Accept — A TCP connection is established with the client
  • Parse → Route → Process → Respond — Transform bytes into a structured reply
The six-state server loop — LISTEN, RECEIVE, ROUTE, PROCESS, SEND, WAIT — cycles endlessly

What Each Step Looks Like

Each step in the loop transforms the request. Parsing turns raw bytes into structured data, routing matches method+path to a handler, and processing runs your business logic.

  • Parsing — Turns raw TCP bytes into a structured request (method, path, headers, body)
  • Routing — Matches method + path to a handler function
  • Processing — Runs business logic: DB queries, auth checks, data transformation
The five-step pipeline processes every request through the same sequential flow

Bare Server vs Framework

A raw server gives you full control but requires you to handle every detail. A framework wraps the plumbing — parsing, routing, error handling — so you focus on writing handlers.

Frameworks handle the plumbing so you focus on business logic

Middleware

Middleware functions sit between the server and your handler. Each layer can inspect, modify, or reject a request before passing it forward.

  • Before — Logging, auth, validation, rate limiting run before your handler
  • After — Response headers, compression, error formatting run on the way back
Middleware layers filter and enrich requests before they reach your handler, then process responses on the way back

The Framework Abstraction

Frameworks like Express, Fastify, and Django wrap the server loop, middleware pipeline, and routing into a clean API. You write handlers. The framework handles everything else — listening, parsing, routing, error handling, serialization.
