The Linux Filesystem

Linux runs about 96% of the world's top one million servers. The filesystem is a single tree rooted at / with standard directories for specific purposes.

  • /home — user home directories for personal files
  • /etc — system configuration files
  • /var — variable data like logs and databases
The Linux filesystem hierarchy — a single tree rooted at / with standard directories for specific purposes

SSH: Secure Shell

SSH (Secure Shell) creates an encrypted tunnel between your machine and a remote server. It is how backend engineers log into production servers to deploy code and troubleshoot issues.

  • ssh user@host — connect to a remote server securely
  • Port 22 is the default SSH port
  • Key-based authentication is more secure than passwords
SSH creates an encrypted tunnel — key exchange, authentication, then a secure session for commands

Permissions: rwx

Every file in Linux has read (r), write (w), and execute (x) permissions for the owner, group, and others. These bits control who can view, modify, or run a file.

  • chmod 755 — owner can write, everyone can read/execute
  • chmod +x script.sh — makes a file executable
  • chown user:group file — changes file ownership
Linux permissions split into owner, group, and others — each with read, write, execute bits
