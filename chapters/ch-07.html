How Systems Talk to Each Other

REST APIs, JSON, URL conventions, GraphQL, and the art of fetching just what you need

~12 min read

REST: Resources + Methods

REST (Representational State Transfer) organizes APIs around resources (nouns) and HTTP methods (verbs). Each resource has a unique URL, and the HTTP method tells the server what action to take on that resource.

The three core principles that make REST predictable and scalable:

  • Resources are nouns — /users, /tweets, /comments. Never verbs like /getUsers. The URL identifies what the resource is.
  • Methods are verbs — GET (read), POST (create), PUT (replace), DELETE (remove). The method says what to do with it.
  • Stateless — each request is independent. The server does not remember past requests. Every request carries all the information needed to process it.
REST organizes APIs around resources (nouns) and HTTP methods (verbs) with a stateless request cycle

The Request-Response Cycle

Every API interaction follows the same pattern: the client sends a request, the server processes it, and the server returns a response. This fundamental cycle powers every web and mobile application.

The request carries a URL, method, headers, and optionally a body. The server validates, applies business logic, queries data, and sends back a status code with a response body — all in milliseconds.

Every API call is a three-step dance: Client → Server → Database → Response

URL Structure Breakdown

A URL is more than just an address — it is a structured message that tells the server exactly what the client wants. Every part has a specific purpose in the HTTP protocol.

Understanding the anatomy of a URL helps you design consistent APIs and debug routing issues faster.

HTTP Headers

Headers carry metadata about the request or response. They tell the server the format of the body, what format the client wants back, who the client is, and how to handle caching. Headers are key-value pairs sent before the body.

Key headers include Authorization (bearer token for authentication), Content-Type (format of the request body), Accept (desired response format), and User-Agent (client identity). Response headers include Set-Cookie, Cache-Control, and Content-Encoding.

JSON Data

JSON (JavaScript Object Notation) is the universal format for API data. It is lightweight, human-readable, and supported by every programming language. A JSON response wraps data in a structured tree of key-value pairs.

JSON has six data types: objects ({}), arrays ([]), strings ("hello"), numbers (42), booleans (true/false), and null. Every API response is a combination of these building blocks.

Pagination: Offset vs Cursor

APIs split large result sets into pages to avoid sending gigabytes of data at once. Two main strategies exist for pagination, each with different trade-offs.

Offset pagination skips N items using OFFSET — simple but fragile when data changes. Cursor pagination uses an opaque token — stable for real-time data but slightly more complex to implement.

Use cursor pagination for real-time data. Offset is fine for static datasets.

The Full Request Lifecycle

From the moment a client sends a request to the moment it receives a response, a full lifecycle unfolds: DNS resolution, TCP connection, TLS handshake, request routing, authentication, business logic, database queries, response serialization, and transmission back to the client.

Each step adds measurable latency. Modern systems optimize every stage — connection pooling, caching, compression, and indexing — to deliver responses in under 200 milliseconds. The API contract decouples the client from the server, allowing each side to evolve independently.
