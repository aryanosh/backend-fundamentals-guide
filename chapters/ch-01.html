How Machines Find Each Other

DNS, IP addresses, ports, and the handshake that starts everything

~10 min read

How Do Computers Even Find Each Other?

Every device on the internet needs a unique address — an IP address — just like a street address for mail. But humans can't remember strings of numbers like 104.244.42.65. That's where DNS (Domain Name System) acts as the internet's phonebook, translating names like google.com into IP addresses in milliseconds.

  • Every device has a unique IP address
  • DNS translates domain names to IPs
  • The lookup takes milliseconds
The DNS lookup: browser queries a resolver, which returns Google's IP address

Ports: The Apartment Number

If an IP address is a building's street address, a port is the apartment number. The standard port for web traffic is port 80 (HTTP) and port 443 (HTTPS). Together, an IP and port form a socket — the complete destination for your data.

  • Port 80 = HTTP, Port 443 = HTTPS
  • Ports 0-1023 are well-known
  • IP + Port = Socket
IP address = building address, port = apartment number. Together they form a socket.

The DNS Resolution Chain

DNS resolution follows a hierarchy. Your query travels through multiple servers before returning the IP address.

  • Recursive resolver → Root → TLD → Authoritative
  • Each level knows where to ask next
  • Results are cached at every level for speed
DNS resolution follows a multi-level hierarchy, with caching at each step

The TCP Three-Way Handshake

TCP is like a phone call — before you start talking, you make sure the other person is ready. The three-way handshake establishes a reliable connection with three messages exchanged in under 100ms.

  • SYN — "Can we talk?"
  • SYN-ACK — "Yes, I'm ready"
  • ACK — "Connection established!"
The TCP three-way handshake: SYN, SYN-ACK, ACK — three messages establish a reliable connection

The Google Example: What Happens When You Open the App

Here's what happens below the surface when you tap Google Search:

  • Check DNS cache → Query resolver → Get IP
  • TCP handshake (SYN, SYN-ACK, ACK)
  • TLS encryption → Send HTTP request → Get response
The full chain from tapping "Search" to seeing results — six steps, under 200ms
